# Design Phase Prompt

## 端到端交付 - Design Phase（架构设计阶段）

### 任务目标
基于需求发现和代码库探索的成果，设计完整的UI/UX架构方案，包括页面布局、组件设计、交互流程和数据接口。

### 已确认的需求
1. **页面整合**: 完全合并为统一首页（workspace作为主入口）
2. **核心功能**: 财务概览卡片 + 活动时间线 + 趋势图表（后续迭代）
3. **目标用户**: 个人理财用户（25-35岁城市白领）
4. **设计风格**: 更简洁现代（扁平化、减少装饰）
5. **移动端**: 暂不需要，专注桌面
6. **数据刷新**: 手动+定时刷新
7. **时间约束**: 1周内MVP

### 代码库探索成果
- ✅ workspace.html已有完整的SPA架构
- ✅ dashboard.py提供统计数据API
- ✅ variables.css提供完整的设计token
- ✅ toast.js等可复用组件
- ✅ Clean Professional Dashboard设计风格

### 设计任务

1. **页面布局设计**
   - Dashboard视图布局（财务概览卡片）
   - 活动时间线布局
   - 侧边栏导航优化
   - 响应式布局规划

2. **组件设计**
   - 财务概览卡片组件
   - 活动时间线组件
   - 数据刷新指示器
   - 加载状态组件

3. **交互设计**
   - 视图切换动画
   - 数据加载反馈
   - 错误状态处理
   - 用户操作流程

4. **数据接口设计**
   - 扩展dashboard API
   - 活动时间线API
   - 数据刷新API

5. **视觉设计**
   - 颜色方案优化
   - 间距和布局规范
   - 字体层级
   - 图标系统

### 设计输出要求

请生成以下设计文档和规范：

1. **页面布局设计图**
   - Dashboard视图布局（ASCII或描述）
   - 活动时间线布局
   - 整体布局结构

2. **组件设计规范**
   - 财务概览卡片（收入、支出、余额、交易数）
   - 活动时间线组件
   - 其他UI组件

3. **交互设计说明**
   - 用户操作流程图
   - 状态转换说明
   - 动画和过渡效果

4. **API接口设计**
   - 扩展的dashboard接口
   - 活动时间线接口
   - 请求/响应格式

5. **视觉规范**
   - 颜色使用规范
   - 间距规范
   - 字体规范
   - 图标使用规范

6. **实施蓝图**
   - 开发任务分解
   - 实施优先级
   - 技术实现要点

请使用高质量的设计方案，生成详细的架构设计报告。

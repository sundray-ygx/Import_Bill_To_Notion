# 实施执行 Prompt - 文档更新与设计优化

## 角色定义
你是一个资深的前端开发和文档专家，负责执行文档更新、页面设计优化和文字内容优化的实施工作。

## 任务背景
基于需求发现、代码库探索和架构设计的结果，需要执行以下实施任务：
1. 创建新的文档（用户指南、设计系统、术语表）
2. 优化现有文档（README、.env.example）
3. 提取和统一CSS样式
4. 优化页面文字内容

## 实施方案
采用**方案B（均衡型优化）**：
- 在保持现有风格的基础上进行适度优化
- 建立轻量级设计系统
- 提取可复用组件

## 实施任务

### Phase 1: 文档优化

#### 1.1 创建 USER_GUIDE.md
```markdown
# 用户使用指南

## 快速开始
## 账单导入流程
## 复盘功能使用
## 常见问题
```

#### 1.2 优化 README.md
- 添加复盘功能亮点
- 更新功能特性列表
- 添加快速开始部分
- 更新截图（如果需要）

#### 1.3 完善 .env.example
- 为每个配置项添加详细注释
- 提供配置示例和默认值说明

#### 1.4 创建 TERMINOLOGY.md
- 功能术语定义
- 技术术语解释
- 使用场景说明

### Phase 2: 页面设计优化

#### 2.1 创建 DESIGN_SYSTEM.md
- 颜色系统说明
- 组件使用指南
- 间距和布局规范

#### 2.2 创建 variables.css
```css
:root {
    /* 颜色系统 */
    --primary-gradient: linear-gradient(135deg, #667eea 0%, #764ba2 100%);
    --primary-color: #667eea;
    --success-color: #10b981;
    --danger-color: #ef4444;
    --warning-color: #f59e0b;
    --info-color: #3b82f6;

    /* 间距系统 */
    --spacing-xs: 0.25rem;
    --spacing-sm: 0.5rem;
    --spacing-md: 1rem;
    --spacing-lg: 1.5rem;
    --spacing-xl: 2rem;

    /* 圆角系统 */
    --radius-sm: 6px;
    --radius-md: 10px;
    --radius-lg: 16px;

    /* 阴影系统 */
    --shadow-sm: 0 1px 2px rgba(0, 0, 0, 0.05);
    --shadow-md: 0 2px 8px rgba(0, 0, 0, 0.08);
    --shadow-lg: 0 4px 16px rgba(0, 0, 0, 0.12);

    /* 过渡效果 */
    --transition-fast: 150ms ease;
    --transition-normal: 250ms ease;
    --transition-slow: 350ms ease;
}
```

#### 2.3 创建 components.css
提取可复用组件样式：
- 按钮组件
- 卡片组件
- 表单组件
- 消息提示组件

#### 2.4 提取内联样式
- 从各页面提取重复样式
- 使用CSS变量替换硬编码值
- 确保样式一致性

#### 2.5 优化响应式设计
- 统一断点定义
- 优化移动端布局
- 测试各种屏幕尺寸

### Phase 3: 文字内容优化

#### 3.1 术语统一
- 使用"复盘"作为功能名称
- 统一按钮文字格式
- 统一页面标题格式

#### 3.2 提示信息优化
- 成功提示：提供后续操作建议
- 错误提示：提供具体解决建议
- 警告提示：提供风险评估

## 实施原则

1. **保持一致性** - 不改变现有的视觉风格
2. **渐进式改进** - 分阶段实施，降低风险
3. **可维护性优先** - 提取可复用组件
4. **用户体验导向** - 优化关键交互流程

### 设计质量

### 文字质量

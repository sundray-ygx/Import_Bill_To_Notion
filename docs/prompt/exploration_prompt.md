# 代码库探索 Prompt - 文档更新与设计优化

## 角色定义
你是一个资深的代码库分析专家，负责对 Notion 账单导入服务进行深入的代码库探索。

## 任务背景
系统已完成账单复盘功能的开发，现在需要深入探索代码库以：
1. 确认所有复盘相关文件的准确位置和状态
2. 分析现有文档的完整性
3. 分析现有设计系统的实现细节
4. 识别需要优化的地方

## 探索目标

### 1. 复盘功能文件映射
确认以下文件的完整性和实现状态：
- `review_service.py` - 复盘服务核心逻辑
- `web_service/routes/review.py` - API 路由
- `web_service/templates/review.html` - 前端页面
- `web_service/static/js/review.js` - JavaScript 逻辑
- `web_service/static/css/review.css` - 样式

### 2. 文档状态分析
分析以下文档的当前状态：
- `README.md` - 主文档
- `CLAUDE.md` - 开发指南
- `.env.example` - 配置示例
- `docs/API_REFERENCE.md` - API 文档

### 3. 设计系统分析
分析现有设计系统的实现：
- CSS 变量定义（颜色、间距、圆角、阴影）
- 组件样式（卡片、按钮、表单）
- 响应式设计实现
- 动画效果

### 4. 导航集成分析
确认复盘功能在以下位置的集成：
- `navbar.html` - 导航栏
- `index.html` - 首页
- `settings.html` - 设置页面

### 5. 术语使用一致性
检查以下位置的术语使用：
- 页面标题
- 按钮文字
- 提示信息
- 错误消息

## 探索方法
使用渐进式检索模式（iterative-retrieval）：
1. **DISPATCH** - 初步探索，识别关键文件
2. **EVALUATE** - 评估发现，确定需要深入探索的区域
3. **REFINE** - 细化探索，深入分析关键部分
4. **LOOP** - 循环 refine，最多 3 次

## 输出要求

### 1. 文件结构映射
提供完整的目录结构树

### 2. 现有功能分析
- 复盘功能实现清单
- API 端点列表
- 数据模型字段

### 3. 设计系统评估
- CSS 变量系统
- 组件样式清单
- 响应式设计状态

### 4. 识别的优化点
- 技术债务清单
- 设计不一致问题
- 文档改进建议

### 5. 术语一致性分析
- 术语对照表
- 一致性检查结果

## 特别说明
- 使用 Read、Glob、Grep 工具进行代码探索
- 提供具体的代码示例和文件位置
- 标注关键文件的优先级（P0/P1/P2）

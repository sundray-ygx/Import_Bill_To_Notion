# 需求发现 Prompt - 文档更新与设计优化

## 角色定义
你是一个资深的需求分析专家，负责对 Notion 账单导入服务的文档更新和设计优化需求进行深度分析。

## 任务背景
系统已完成账单复盘功能的开发，现在需要：
1. 更新相关文档(*.md)以反映完整的账单管理功能
2. 对系统页面做全面的设计优化
3. 更新页面文字以匹配新功能

## 系统现状
根据 CLAUDE.md，系统具备以下功能：
- 用户认证（注册、登录、密码管理）
- 多用户支持与数据隔离
- 账单上传和导入（支付宝、微信、银联）
- 账单历史记录管理
- **账单复盘功能**（月度、季度、年度财务复盘）
- 管理员功能（用户管理、审计日志）
- Notion 配置验证

## 分析维度

### 1. 文档更新需求
- 需要更新哪些 .md 文件？
- 更新重点是什么？

### 2. 页面设计需求
- 主要关注哪些页面？
- 设计目标是什么？
- 是否需要响应式设计？

### 3. 文字更新需求
- 哪些页面的文字需要更新？
- 期望的语气和风格？

## 输出要求

### 1. 需求澄清问题列表
提供至少10个需要用户确认的问题

### 2. 复杂度评估
从以下四个维度评估：
- 功能复杂度 (1-10)
- 技术复杂度 (1-10)
- 规模复杂度 (1-10)
- 风险复杂度 (1-10)

### 3. 开发模式建议
根据复杂度选择：瀑布流 or 敏捷

### 4. 实施建议
提供详细的实施计划，包括：
- 任务拆解（Epic -> Feature -> Story）
- 验收标准
- 风险与缓解措施

## 特别说明
- 忽略之前生成的各阶段文档
- 结合新需求重新生成
- 考虑本地模板融合（如果存在）

# 质量验证 Prompt - 文档更新与设计优化

## 角色定义
你是一个资深的QA专家，负责对文档更新和设计优化的实施结果进行全面的质量验证。

## 验证背景
已完成文档更新、页面设计优化和文字内容优化的实施工作，现在需要进行质量验证。

## 验证范围

### 1. 文档验证
- 新创建的文档
- 更新的现有文档
- 文档链接有效性
- 示例代码正确性

### 2. 设计验证
- CSS变量系统
- 组件样式一致性
- 响应式布局
- 浏览器兼容性

### 3. 文字验证
- 术语使用一致性
- 拼写和语法检查
- 提示信息质量
- 用户理解度

## 验证标准

### 文档验证标准

#### 新文档检查
- [ ] USER_GUIDE.md 存在且内容完整
- [ ] DESIGN_SYSTEM.md 存在且内容完整
- [ ] TERMINOLOGY.md 存在且内容完整

#### 更新文档检查
- [ ] README.md 已更新复盘功能说明
- [ ] .env.example 已添加配置注释
- [ ] API_REFERENCE.md 已完善复盘API

#### 质量检查
- [ ] 所有文档链接有效
- [ ] 示例代码可以运行
- [ ] 配置说明清晰易懂

### 设计验证标准

#### CSS变量系统
- [ ] variables.css 文件存在
- [ ] 所有必要的变量已定义
- [ ] 变量命名规范统一

#### 组件样式
- [ ] components.css 文件存在
- [ ] 所有组件样式已提取
- [ ] 组件样式使用CSS变量

#### 一致性检查
- [ ] 所有页面使用统一的颜色系统
- [ ] 所有页面使用统一的间距系统
- [ ] 所有页面使用统一的圆角系统

#### 响应式设计
- [ ] 移动端布局正常
- [ ] 平板布局正常
- [ ] 桌面布局正常
- [ ] 断点定义统一

### 文字验证标准

#### 术语一致性
- [ ] "复盘"术语使用统一
- [ ] 按钮文字格式统一
- [ ] 页面标题格式统一

#### 提示信息质量
- [ ] 成功提示友好且有帮助
- [ ] 错误提示提供解决建议
- [ ] 警告提示提供风险评估

#### 语言质量
- [ ] 无拼写错误
- [ ] 无语法错误
- [ ] 语句通顺易懂

## 验证方法

### 1. 文档验证
```bash
# 检查文档文件存在
ls -la docs/USER_GUIDE.md
ls -la docs/DESIGN_SYSTEM.md
ls -la docs/TERMINOLOGY.md

# 检查链接有效性
# （需要手动验证或使用工具）

# 检查示例代码
# （需要手动运行测试）
```

### 2. 设计验证
```bash
# 检查CSS文件存在
ls -la web_service/static/css/variables.css
ls -la web_service/static/css/components.css

# 检查CSS变量定义
grep "var(--" web_service/static/css/*.css

# 检查组件样式
grep "\.btn\|\.card\|\.form" web_service/static/css/components.css
```

### 3. 文字验证
```bash
# 检查术语使用
grep -r "复盘" web_service/templates/
grep -r "报告\|分析" web_service/templates/

# 拼写检查
# （需要使用拼写检查工具）
```

## 问题分级

### 严重问题 (P0)
- 阻塞功能使用的问题
- 数据丢失风险
- 安全漏洞

### 重要问题 (P1)
- 影响用户体验的问题
- 设计不一致明显
- 文档错误或遗漏

### 一般问题 (P2)
- 优化建议
- 代码风格问题
- 文档完善建议

## 输出要求

1. 完整的质量验证报告
2. 问题清单（按优先级排序）
3. 修复建议和行动计划

## 特别说明
- 使用 Read、Grep、Bash 工具进行验证
- 提供具体的问题证据
- 给出可操作的修复建议
